SharePoint Online Essential Tools: Complete Administrator's Guide

By Nagendra Jayaram • September 15, 2025 • 10 min read
SharePoint Online Tools Microsoft 365 PowerShell Administration

As a Microsoft Sr Technical Advisor with 18+ years of enterprise experience, I've worked with virtually every SharePoint Online administrative tool available. The right tools can make the difference between hours of frustration and minutes of efficient problem resolution.

This comprehensive guide covers the five essential tools every SharePoint Online administrator and developer should master. These tools form the foundation of effective SharePoint administration and are indispensable for maintaining enterprise-level environments.

1. GitHub Search Query Tool

The GitHub Search Query Tool is an enhanced version of SharePoint's native search tool that provides comprehensive KQL testing and debugging capabilities. This community-maintained tool offers advanced features for testing search functionality, debugging query issues, and analyzing search behavior in SharePoint Online environments. It includes better error handling, enhanced result visualization, and additional debugging options not available in the standard SharePoint tool. The tool is actively maintained and regularly updated with new features based on community feedback.

Access the Tool

GitHub Repository: SharePoint Search Query Tool

Download and host locally or use the web-based version for immediate testing.

Usage Examples

# Test document library content ContentClass:STS_ListItem_DocumentLibrary # Search by author and file type Author:"John Smith" AND FileExtension:pdf # Find recently modified content LastModifiedTime>=2025-09-01 # Search specific site paths Path:"https://contoso.sharepoint.com/sites/marketing*"

2. SharePoint Client Browser

SharePoint Client Browser is a powerful desktop application that provides low-level access to SharePoint's object model for advanced administrative tasks. It allows direct manipulation of SharePoint objects and provides hierarchical navigation through site collections, webs, lists, and items. The tool is essential for investigating complex permission scenarios, analyzing field schemas, and performing detailed object model operations. Use with caution as it provides direct access to SharePoint's underlying structure.

Download and Setup

GitHub Repository: SharePoint Client Browser

Download the latest release and install SharePoint Client Components SDK.

How to Use

  1. Install and Launch: Download from GitHub and run the application
  2. Connect to Site: Enter your SharePoint Online site URL and authenticate
  3. Navigate Objects: Browse through the hierarchical tree structure (Site > Webs > Lists > Items)
  4. Inspect Properties: Select any object to view its properties and methods in the right panel
  5. Execute Operations: Use read-only mode for investigation, modify mode for changes (with extreme caution)

Safety Warning

Always use read-only mode for investigation. Test all operations in development environments before applying to production. Keep detailed documentation of any changes made.

3. Network Traffic Analysis Tools

Network traffic analysis is crucial for debugging SharePoint Online issues, understanding API calls, and monitoring authentication flows. Use Fiddler for advanced debugging or browser developer tools for basic network analysis.

Download Fiddler

Official Download: Telerik Fiddler Classic or Fiddler Everywhere

Choose Fiddler Classic for Windows or Fiddler Everywhere for cross-platform support.

Capturing Network Traces

Method 1: Using Fiddler

  1. Install Fiddler: Download and install from the official website
  2. Configure HTTPS: Tools > Options > HTTPS > Check "Capture HTTPS CONNECTs" and "Decrypt HTTPS traffic"
  3. Install Certificate: Click "Install Certificate" and trust the Fiddler root certificate
  4. Set Filters: Apply filters for SharePoint domains (*.sharepoint.com, *.microsoftonline.com)
  5. Start Capture: Reproduce the issue while Fiddler captures all HTTP/HTTPS traffic

Method 2: Browser Developer Tools (Alternative)

  1. Open Developer Tools: Press F12 in your browser
  2. Navigate to Network Tab: Click on the "Network" tab
  3. Enable Recording: Ensure recording is enabled (red circle)
  4. Clear Existing: Click clear button to remove old entries
  5. Reproduce Issue: Perform the action that causes the problem
  6. Analyze Results: Review failed requests (red entries) and response codes

SharePoint Correlation ID

Correlation ID Importance

Critical for Support: Every SharePoint Online request generates a unique Correlation ID that Microsoft support uses to trace issues in their logs. Always capture this ID when reporting problems - it's found in error messages, HTTP response headers (SPRequestGuid), or ULS logs.

Common HTTP Response Codes in SharePoint

# Success Codes 200 OK - Request successful 201 Created - Resource created successfully 204 No Content - Successful DELETE operation # Client Error Codes 400 Bad Request - Invalid request syntax 401 Unauthorized - Authentication required 403 Forbidden - Access denied (permissions) 404 Not Found - Resource doesn't exist 409 Conflict - Resource already exists 413 Payload Too Large - File size exceeds limits # Server Error Codes 500 Internal Server Error - General server error 503 Service Unavailable - Service temporarily down 504 Gateway Timeout - Request timeout

Useful Filters for SharePoint Analysis

# Fiddler Filters Host: *.sharepoint.com Host: *.microsoftonline.com Host: *.office.com # Browser DevTools Filters - Filter by domain: sharepoint.com - Filter by status: 4xx, 5xx (errors only) - Filter by method: POST, PUT, DELETE

4. SharePoint Online Management Shell

SharePoint Online Management Shell provides comprehensive PowerShell-based administrative capabilities for bulk operations, automation, and advanced configuration tasks. It's the most powerful tool for SharePoint Online administration, offering both traditional SPO cmdlets and modern PnP PowerShell modules. The tool supports modern authentication with MFA and provides extensive automation capabilities for enterprise environments. PowerShell is essential for tasks that cannot be performed through the web interface.

Connection Steps

# Step 1: Install Required Modules Install-Module -Name Microsoft.Online.SharePoint.PowerShell -Force Install-Module -Name PnP.PowerShell -Force # Step 2: Connect to SharePoint Online Admin Center Connect-SPOService -Url https://[tenant]-admin.sharepoint.com # Step 3: Connect to Specific Site with PnP (Modern) Connect-PnPOnline -Url https://[tenant].sharepoint.com/sites/[sitename] -Interactive # Step 4: Verify Connection Get-SPOSite -Limit 5 # Test connection with admin cmdlets Get-PnPWeb # Test connection with PnP cmdlets

Quick Command Examples

# Get site information Get-SPOSite -Identity https://contoso.sharepoint.com/sites/teamsite # Create new site collection New-SPOSite -Url https://contoso.sharepoint.com/sites/newsite -Owner admin@contoso.com -StorageQuota 1000 # Request site reindexing Request-SPOReIndexWeb -Identity https://contoso.sharepoint.com/sites/teamsite # Manage site permissions Set-SPOUser -Site https://contoso.sharepoint.com/sites/teamsite -LoginName user@contoso.com -IsSiteCollectionAdmin $true

5. Self-Diagnostics Before Contacting Support

Before reaching out to Microsoft Support, administrators should perform these self-diagnostic steps to identify and potentially resolve SharePoint Online issues independently. These diagnostic procedures can save time and often resolve common problems without requiring escalation.

Pre-Support Diagnostic Checklist

1. Service Health Verification

  • Check Service Health: Visit Microsoft 365 Admin Center > Health > Service Health to verify no known outages
  • Review Message Center: Check for recent service updates or planned maintenance
  • Validate Tenant Status: Ensure your tenant is in good standing with no billing issues

2. User Account and Permissions Analysis

# PowerShell commands to check user status Connect-SPOService -Url https://[tenant]-admin.sharepoint.com # Verify user exists and has proper licenses Get-SPOUser -Site https://[tenant].sharepoint.com/sites/[site] -LoginName user@domain.com # Check site collection administrators Get-SPOSite -Identity https://[tenant].sharepoint.com/sites/[site] | Select Owners # Validate external sharing settings Get-SPOSite -Identity https://[tenant].sharepoint.com/sites/[site] | Select SharingCapability

3. Site and Content Validation

  • Test Different Browsers: Verify issue occurs in multiple browsers (Edge, Chrome, Firefox)
  • Incognito/Private Mode: Test in private browsing to rule out cache/cookie issues
  • Different Networks: Try accessing from different network connections
  • Mobile vs Desktop: Test on different device types

4. Search and Indexing Issues

# Force site reindexing if search results are missing Connect-SPOService -Url https://[tenant]-admin.sharepoint.com Request-SPOReIndexWeb -Identity https://[tenant].sharepoint.com/sites/[site] # Check crawl status and last crawl time Get-SPOSite -Identity https://[tenant].sharepoint.com/sites/[site] -Detailed | Select LastContentModifiedDate

5. Storage and Quota Analysis

# Check site storage usage and quotas $site = Get-SPOSite -Identity https://[tenant].sharepoint.com/sites/[site] -Detailed Write-Host "Storage Used: $($site.StorageUsageCurrent) MB" Write-Host "Storage Quota: $($site.StorageQuota) MB" Write-Host "Percentage Used: $([math]::Round(($site.StorageUsageCurrent/$site.StorageQuota)*100,2))%" # Identify large lists that might cause performance issues Connect-PnPOnline -Url https://[tenant].sharepoint.com/sites/[site] -Interactive Get-PnPList | Where-Object {$_.ItemCount -gt 5000} | Select Title, ItemCount

Documentation for Support

If self-diagnostics don't resolve the issue, gather this information before contacting support:

  • Correlation ID: From error messages or HTTP headers (SPRequestGuid)
  • Exact Error Messages: Screenshots and full text of errors
  • Reproduction Steps: Detailed steps to reproduce the issue
  • User Impact: How many users affected and business impact
  • Environment Details: Browser versions, network setup, device types
  • Recent Changes: Any recent configuration changes or updates

Common Issue Quick Fixes

# Fix: Site User ID Mismatch (similar to Microsoft Learn article) # This resolves issues where user permissions appear correct but access is denied # Step 1: Remove user from site Remove-SPOUser -Site https://[tenant].sharepoint.com/sites/[site] -LoginName user@domain.com # Step 2: Clear any cached permissions Start-Sleep -Seconds 30 # Step 3: Re-add user with proper permissions Add-SPOUser -Site https://[tenant].sharepoint.com/sites/[site] -LoginName user@domain.com -Group "Members" # Fix: Search Not Working Request-SPOReIndexWeb -Identity https://[tenant].sharepoint.com/sites/[site] # Fix: External Sharing Issues Set-SPOSite -Identity https://[tenant].sharepoint.com/sites/[site] -SharingCapability ExternalUserSharingOnly

Tool Mastery Strategy

Mastering these five essential tools requires hands-on practice and understanding when to apply each one. Start with the Search Query Tool and PowerShell Management Shell for daily administration tasks, then gradually incorporate Fiddler and Client Browser for more complex scenarios.

The key to becoming proficient is regular practice in safe environments. Set up a dedicated development tenant where you can experiment with these tools without risk to production systems.

Recommended Learning Path

  1. Start with PowerShell: Master basic cmdlets and common administrative tasks
  2. Learn Search Query Tool: Understand KQL and search mechanics
  3. Practice with Browser Tools: Use F12 tools for client-side diagnostics
  4. Advance to Fiddler: Master HTTP debugging and API analysis
  5. Explore Client Browser: Use for complex object model scenarios
