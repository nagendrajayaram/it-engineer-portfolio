1. SharePoint Search Query Tool

The SharePoint Search Query Tool is an invaluable browser-based utility for testing and troubleshooting search functionality in SharePoint Online. It provides direct access to the search API without requiring custom code.

Key Features and Capabilities

  • Query Testing: Test KQL (Keyword Query Language) and search queries in real-time
  • Result Analysis: View detailed search results including metadata and relevance scores
  • Query Suggestions: Test auto-complete and query suggestion functionality
  • People Search: Specialized interface for testing people and expertise search

How to Access and Use

  1. Navigate to your SharePoint Online site
  2. Append /_layouts/15/searchquerytool.aspx to your site URL
  3. Enter your KQL query in the query text box
  4. Configure additional parameters (select properties, result count, etc.)
  5. Execute the query and analyze results
https://contoso.sharepoint.com/sites/yoursite/_layouts/15/searchquerytool.aspx

Common Search Issues to Troubleshoot:

  • Content not appearing in search: Check crawl status and managed properties
  • Incorrect ranking: Analyze relevance scores and query processing
  • Performance issues: Test query response times and optimize KQL
  • People search problems: Verify user profile synchronization

Advanced Query Examples

# Search for documents by author Author:"John Smith" ContentClass:STS_ListItem_DocumentLibrary # Find pages modified in last week LastModifiedTime>=2025-09-08 ContentClass:STS_Web # Search specific file types FileExtension:pdf OR FileExtension:docx

2. SharePoint Client Browser

The SharePoint Client Browser is a desktop application that provides low-level access to SharePoint's object model, allowing administrators to browse, inspect, and modify SharePoint objects directly.

Core Functionality

  • Object Model Navigation: Browse through site collections, webs, lists, and items
  • Property Inspection: View and edit object properties in real-time
  • Method Execution: Execute SharePoint object methods directly
  • Batch Operations: Perform bulk operations on multiple objects

Installation and Setup

  1. Download SharePoint Client Browser from CodePlex or GitHub
  2. Install on a machine with SharePoint Client Components
  3. Launch and connect using your SharePoint Online credentials
  4. Authenticate using modern authentication (OAuth)

When to Use Client Browser:

  • Permission Issues: Investigate complex permission inheritance problems
  • Content Type Problems: Debug content type provisioning and inheritance
  • List Schema Issues: Examine field definitions and list configurations
  • Workflow Debugging: Inspect workflow associations and status

Best Practices for Safe Usage

  • Always test in a development environment first
  • Take screenshots/notes before making changes
  • Understand the impact of modifications before executing
  • Use read-only mode when possible for investigation

3. Network Capture with Fiddler

Fiddler is a powerful HTTP debugging proxy that captures all HTTP(S) traffic between your computer and SharePoint Online. It's essential for troubleshooting authentication, API calls, and performance issues.

Setup for SharePoint Online

  1. Download and install Fiddler Classic or Fiddler Everywhere
  2. Configure HTTPS decryption in Tools > Options > HTTPS
  3. Install Fiddler certificate for HTTPS traffic inspection
  4. Configure browser proxy settings if needed

Key Features for SharePoint Troubleshooting

  • Request/Response Analysis: Inspect HTTP headers, payloads, and status codes
  • Authentication Debugging: Monitor OAuth flows and token exchanges
  • Performance Monitoring: Identify slow requests and bottlenecks
  • API Call Tracking: Monitor REST API and GraphQL requests
# Useful Fiddler Filters for SharePoint Host: *.sharepoint.com Host: *.microsoftonline.com Method: POST Status: 4** Status: 5**

Common Issues Fiddler Helps Diagnose:

  • Authentication Failures: Track OAuth redirects and token validation
  • CORS Issues: Identify cross-origin request problems
  • API Errors: Capture detailed error responses from REST endpoints
  • Performance Problems: Measure request timing and identify slow operations
  • Upload Issues: Debug file upload failures and size limits

Advanced Fiddler Techniques

  • AutoResponder: Mock responses for testing scenarios
  • Composer: Craft custom HTTP requests for API testing
  • Rules: Automatically modify requests/responses
  • Extensions: Use specialized add-ons for SharePoint analysis

4. SharePoint Online Management Shell

SharePoint Online Management Shell is a PowerShell-based command-line tool that provides comprehensive administrative capabilities for SharePoint Online environments.

Installation and Connection

# Install SharePoint Online Module Install-Module -Name Microsoft.Online.SharePoint.PowerShell # Connect to SharePoint Online Connect-SPOService -Url https://contoso-admin.sharepoint.com # Modern authentication with MFA support Connect-SPOService -Url https://contoso-admin.sharepoint.com -ModernAuth $true

Essential Troubleshooting Cmdlets

Site Collection Management

# Get site collection information Get-SPOSite -Identity https://contoso.sharepoint.com/sites/teamsite # Check site collection storage and quota Get-SPOSite -Detailed | Select Url, StorageUsageCurrent, StorageQuota # Resolve site collection issues Repair-SPOSite -Identity https://contoso.sharepoint.com/sites/teamsite

User and Permission Management

# Get site collection administrators Get-SPOSiteGroup -Site https://contoso.sharepoint.com/sites/teamsite # Add external user Add-SPOUser -Site https://contoso.sharepoint.com/sites/teamsite -LoginName user@external.com -Group "Members" # Check sharing settings Get-SPOSite -Identity https://contoso.sharepoint.com/sites/teamsite | Select SharingCapability

Search and Content Management

# Trigger site reindexing Request-SPOReIndexWeb -Identity https://contoso.sharepoint.com/sites/teamsite # Check search schema Get-SPOSearchConfiguration -SearchObjectLevel SPSiteSubscription # Export search configuration Export-SPOSearchConfiguration -Owner "tenant"

PowerShell Troubleshooting Scenarios:

  • Bulk Permission Changes: Script permission modifications across multiple sites
  • Site Provisioning Issues: Automate site creation and configuration
  • Content Migration: Migrate content between site collections
  • Compliance and Auditing: Generate reports on site usage and permissions
  • Search Configuration: Manage search schemas and result sources

Modern PowerShell with PnP

# Install PnP PowerShell Install-Module PnP.PowerShell # Connect with modern authentication Connect-PnPOnline -Url https://contoso.sharepoint.com/sites/teamsite -Interactive # Advanced operations Get-PnPList | Where-Object {$_.Hidden -eq $false} Get-PnPFile -Url "/sites/teamsite/Shared Documents/document.pdf"

5. Self-Diagnostics and Health Checks

SharePoint Online includes built-in diagnostic capabilities and health monitoring tools that provide insights into system performance and identify potential issues.

Microsoft 365 Admin Center Diagnostics

  • Service Health Dashboard: Monitor service incidents and advisories
  • Message Center: Stay informed about planned changes and updates
  • Usage Reports: Analyze SharePoint usage patterns and performance
  • Security & Compliance: Monitor security incidents and compliance status

SharePoint Health Analyzer

# Access via SharePoint Admin Center https://[tenant]-admin.sharepoint.com/_layouts/15/online/AdminHome.aspx # Key areas to monitor: - Site collection health - Storage utilization - Sharing and external access - Search health and performance

Browser-Based Diagnostics

  • Developer Tools: Use F12 tools for client-side debugging
  • Network Tab: Monitor request timing and failures
  • Console Errors: Identify JavaScript errors and warnings
  • Application Insights: Leverage telemetry for performance monitoring

Self-Diagnostic Checklist:

  • Performance Baselines: Establish normal performance metrics
  • Regular Health Checks: Schedule periodic system assessments
  • User Experience Monitoring: Track page load times and user interactions
  • Proactive Alerting: Set up notifications for critical issues
  • Capacity Planning: Monitor storage and usage trends

Custom Monitoring Solutions

# PowerShell script for regular health checks $sites = Get-SPOSite -Limit All foreach ($site in $sites) { $health = Get-SPOSite -Identity $site.Url -Detailed if ($health.StorageUsageCurrent -gt ($health.StorageQuota * 0.8)) { Write-Warning "Site $($site.Url) is approaching storage limit" } }

Common Troubleshooting Scenarios

Scenario 1: Search Results Not Appearing

Step-by-Step Resolution:

  1. Use Search Query Tool: Test if content is indexed using KQL queries
  2. Check Fiddler: Monitor search API calls for errors
  3. PowerShell Verification: Use Request-SPOReIndexWeb to force reindexing
  4. Verify Permissions: Ensure search crawler has access to content

Scenario 2: Authentication and Access Issues

Diagnostic Approach:

  1. Fiddler Analysis: Capture OAuth flow and identify failure points
  2. Client Browser: Inspect user permissions and group memberships
  3. PowerShell Checks: Verify external sharing settings and user status
  4. Admin Center Review: Check security and compliance policies

Scenario 3: Performance and Loading Issues

Performance Investigation:

  1. Browser DevTools: Analyze page load waterfall and identify bottlenecks
  2. Fiddler Timing: Measure server response times and request sizes
  3. Health Diagnostics: Review site collection health and storage usage
  4. PowerShell Optimization: Implement caching and optimize queries

Security Considerations

  • Principle of Least Privilege: Use minimal permissions for diagnostics
  • Secure Tool Access: Protect credentials and access tokens
  • Audit Trail: Maintain records of administrative actions
  • Compliance Awareness: Consider regulatory requirements when using tools

Continuous Improvement

  • Stay Updated: Keep tools and modules current with latest versions
  • Share Knowledge: Document solutions for team reference

  • Monitor Trends: Track recurring issues and implement preventive measures
